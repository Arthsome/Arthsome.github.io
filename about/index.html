Disclosure Policy

CVE reports and vulnerability analyses are published only after an official vendor patch or responsible disclosure process is complete. No exploit code or working payloads are ever included. If something I've published looks problematic, reach out before filing a takedown.

How to Add New Posts

This blog is a plain static site on GitHub Pages — no build tool or framework required. To publish a new post, create a folder under posts/your-slug/, write an index.html based on the CVE post template, set the correct category pill (category-pill cve or category-pill writeup), then add a post card to posts/index.html, the relevant category page, and optionally the homepage. Push to main and GitHub Pages deploys automatically.
